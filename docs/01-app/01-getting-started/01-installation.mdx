Hozz létre egy új Next.js alkalmazást, és futtasd helyben.

## Gyors kezdés

1. Hozz létre egy új Next.js alkalmazást `my-app` néven.
2. Futtasd a `cd my-app` parancsot, és indítsd el a fejlesztői szervert.
3. Látogass el a `http://localhost:3000` címre.

```bash package="pnpm"
pnpm create next-app@latest my-app --yes
cd my-app
pnpm dev
```

```bash package="npm"
npx create-next-app@latest my-app --yes
cd my-app
npm run dev
```

```bash package="yarn"
yarn create next-app@latest my-app --yes
cd my-app
yarn dev
```

```bash package="bun"
bun create next-app@latest my-app --yes
cd my-app
bun dev
```

- A `--yes` opció kihagyja a kezdő promptokat és a mentett beállításokat vagy az alapértelmezett beállításokat használja. Az alapértelmezett beállítás engedélyezi a TypeScript, Tailwind, App Router és Turbopack használatát valamint `@/*` import aliast.

</AppOnly>

## Rendszer  követelmények

Mielőtt elkezdenéd, győződj meg róla, hogy a rendszered megfelel a következő követelményeknek:

- [Node.js 18.18](https://nodejs.org/) vagy újabb.
- macOS, Windows (beleértve a WSL) vagy Linux.

## Létrehozás a parancssori felülettel

Egy új Next.js alkalmazás létrehozásának leggyorsabb módja a [`create-next-app`](/docs/app/api-reference/cli/create-next-app) használata, amely mindent automatikusan beállít. Projekt létrehozásához futtassa a következő parancsot:

```bash filename="Terminal"
npx create-next-app@latest
```

A telepítés során a következő utasításokat fogja látni:

```txt filename="Terminal"
What is your project named? my-app                                              // Mi a projekted neve? my-app
Would you like to use TypeScript? No / Yes                                      // TypeScriptet szeretnél használni? Nem / Igen
Which linter would you like to use? ESLint / Biome / None                       // Melyik lintert szeretnéd használni? ESLint / Biome / Nincs
Would you like to use Tailwind CSS? No / Yes                                    // Tailwind CSS-t szeretnéd használni? Nem / Igen
Would you like your code inside a `src/` directory? No / Yes                    // A kódodat egy `src/` könyvtárban szeretnéd látni? Nem / Igen
Would you like to use App Router? (recommended) No / Yes                        // App Routert szeretnéd használni? (ajánlott) Nem / Igen
Would you like to use Turbopack? (recommended) No / Yes                         // Turbopackot szeretnél használni? (ajánlott) Nem / Igen
Would you like to customize the import alias (`@/*` by default)? No / Yes       // Testreszabod az import aliast (alapértelmezés szerint `@/*`)? Nem / Igen
What import alias would you like configured? @/*                                // Milyen import aliast szeretnéd konfigurálni? @/*
```

A promptok után a [`create-next-app`](/docs/app/api-reference/cli/create-next-app) létrehoz egy mappát a projekted nevével, és telepíti a szükséges függőségeket.

## Manual telepítés

Új Next.js alkalmazás manuális létrehozásához telepítse a szükséges csomagokat:

```bash package="pnpm"
pnpm i next@latest react@latest react-dom@latest
```

```bash package="npm"
npm i next@latest react@latest react-dom@latest
```

```bash package="yarn"
yarn add next@latest react@latest react-dom@latest
```

```bash package="bun"
bun add next@latest react@latest react-dom@latest
```

> **Jó tudni**: Az App Router beépített [React canary kiadásokat](https://react.dev/blog/2023/05/03/react-canaries) használ, amelyek tartalmazzák az összes stabil React 19 változást, valamint az újabb, keretrendszerekben validált funkciókat is. A Pages Router a `package.json` fájlban telepített React verziót használja.

Ezután add hozzá a következő szkripteket a `package.json` fájlodhoz:

```json filename="package.json"
{
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "lint": "eslint",
    "lint:fix": "eslint --fix"
  }
}
```

Ezek a parancsok az alkalmazás fejlesztésének különböző szakaszaira vonatkoznak:

- `next dev`: Elindítja a fejlesztői szervert a Turbopack (alapértelmezett csomagoló) használatával.
- `next build`: Felépíti az alkalmazást éles környezethez.
- `next start`: Futtatja az alkalmazást éles környezetben.
- `eslint`: Futtatja az ESLint-et.


A Turbopack mostantól az alapértelmezett csomagoló. A Webpack használatához futtassa a `next dev --webpack` vagy a `next build --webpack` parancsot. A konfigurációs részletekért lásd a [Turbopack dokumentációját](/docs/app/api-reference/turbopack).

### Hozd létre az `app` könyvtárat

A Next.js fájlrendszer-útválasztást használ, ami azt jelenti, hogy az alkalmazásodban lévő útvonalakat a fájlok strukturálása határozza meg.

Hozz létre egy `app` mappát. Ezután az `app` mappán belül hozz létre egy `layout.tsx` fájlt. Ez a fájl a [root layout](/docs/app/api-reference/file-conventions/layout#root-layout). A fájl kötelező, és tartalmaznia kell a `<html>` és `<body>` címkéket.


```tsx filename="app/layout.tsx" switcher
export default function RootLayout({
  children,
}: {
  children: React.ReactNode
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

```jsx filename="app/layout.js" switcher
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  )
}
```

Hozz létre egy `app/page.tsx` kezdőlapot a következő kezdeti tartalommal:

```tsx filename="app/page.tsx" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

```jsx filename="app/page.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

Mind a `layout.tsx`, mind a `page.tsx` fájlok megjelennek, amikor a felhasználó az alkalmazás gyökeréhez (`/`) navigál.

### Hozd létre a `pages` könyvtárat

A Next.js fájlrendszer-útválasztást használ, ami azt jelenti, hogy az alkalmazásodban lévő útvonalakat a fájlok strukturálása határozza meg.

Hozz létre egy `pages` könyvtárat a projekted gyökerében. Ezután adj hozzá egy `index.tsx` fájlt a `pages` mappádban. Ez lesz a kezdőlapod (`/`):
```tsx filename="pages/index.tsx" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

```jsx filename="pages/index.js" switcher
export default function Page() {
  return <h1>Hello, Next.js!</h1>
}
```

Ezután adj hozzá egy `_app.tsx` fájlt a `pages/` könyvtáron belül a globális elrendezés meghatározásához. Tudj meg többet az [egyéni alkalmazásfájlról](/docs/pages/building-your-application/routing/custom-app).

```tsx filename="pages/_app.tsx" switcher
import type { AppProps } from 'next/app'

export default function App({ Component, pageProps }: AppProps) {
  return <Component {...pageProps} />
}
```

```jsx filename="pages/_app.js" switcher
export default function App({ Component, pageProps }) {
  return <Component {...pageProps} />
}
```

Végül adj hozzá egy `_document.tsx` fájlt a `pages/` könyvtáron belül a szervertől érkező kezdeti válasz szabályozásához. Tudj meg többet az [egyéni dokumentumfájlról](/docs/pages/building-your-application/routing/custom-document).

```tsx filename="pages/_document.tsx" switcher
import { Html, Head, Main, NextScript } from 'next/document'

export default function Document() {
  return (
    <Html>
      <Head />
      <body>
        <Main />
        <NextScript />
      </body>
    </Html>
  )
}
```

```jsx filename="pages/_document.js" switcher
import { Html, Head, Main, NextScript } from 'next/document'

export default function Document() {
  return (
    <Html>
      <Head />
      <body>
        <Main />
        <NextScript />
      </body>
    </Html>
  )
}
```

### Hozd létre a „public” mappát (opcionális)

Hozz létre egy [`public` mappát](/docs/app/api-reference/file-conventions/public-folder) a projekted gyökerében, ahol statikus elemeket, például képeket, betűtípusokat stb. tárolhatsz. A `public` mappában található fájlokra ezután a kódod hivatkozhat az alap URL-ből (`/`) kiindulva.


Ezekre az elemekre ezután a gyökérútvonal (`/`) segítségével hivatkozhatsz. Például a `public/profile.png` fájlra `/profile.png` néven lehet hivatkozni:

```tsx filename="app/page.tsx" highlight={4} switcher
import Image from 'next/image'

export default function Page() {
  return <Image src="/profile.png" alt="Profile" width={100} height={100} />
}
```

```jsx filename="app/page.js" highlight={4} switcher
import Image from 'next/image'

export default function Page() {
  return <Image src="/profile.png" alt="Profile" width={100} height={100} />
}
```

## Futtassa a fejlesztői szervert

1. Futtassd az `npm run dev` parancsot a fejlesztői szerver elindításához.
2. Látogass el a `http://localhost:3000` címre az alkalmazás megtekintéséhez.
3. Szerkeszd az <AppOnly>`app/page.tsx`</AppOnly><PagesOnly>`pages/index.tsx`</PagesOnly> fájlt, és mentsd el, hogy a frissített eredményt a böngészőben is láthasd.

## TypeScript beállítása

> Minimális TypeScript verzió: `v5.1.0`

A Next.js beépített TypeScript támogatással rendelkezik. TypeScript hozzáadásához a projektedhez nevezd át a fájlt `.ts` / `.tsx` kiterjesztésűre, és futtasd a `next dev` parancsot. A Next.js automatikusan telepíti a szükséges függőségeket, és hozzáad egy `tsconfig.json` fájlt az ajánlott konfigurációs beállításokkal.

## Kódellenőrzés konfigurálása

A Next.js támogatja a kódellenőrzést az ESLint vagy a Biome használatával. Válasssz ki egy lintert, és futtasd közvetlenül a `package.json` szkripteken keresztül.

- Használj **ESLint**-et (átfogó szabályok):

```json filename="package.json"
{
  "scripts": {
    "lint": "eslint",
    "lint:fix": "eslint --fix"
  }
}
```

- Vagy használd a **Biome**-ot (gyors kódellenőrző + formázó):

```json filename="package.json"
{
  "scripts": {
    "lint": "biome check",
    "format": "biome format --write"
  }
}
```

Ha a projekted korábban a `next lint` függvényt használta, akkor migráld a szkripteket az ESLint CLI-be a következő kódmóddal:

```bash filename="Terminal"
npx @next/codemod@canary next-lint-to-eslint-cli .
```

987 / 5 000
Ha ESLint-et használsz, hozz létre egy explicit konfigurációt (ajánlott: `eslint.config.mjs`). Az ESLint mind a [régi `.eslintrc.*`, mind az újabb `eslint.config.mjs` formátumokat támogatja](https://eslint.org/docs/latest/use/configure/configuration-files#configuring-eslint). Az ajánlott beállításért lásd az [ESLint API referenciát](/docs/app/api-reference/config/eslint#with-core-web-vitals).

> **Jó tudni**: A Next.js 16 előtt a `next build` automatikusan futtatta az ellenőrzést. Most ezt az automatikus, build-idejű kódellenőrzést eltávolítottuk. A kódellenőrzés futtatásának szabályozásához hívd meg a lintinget npm szkripteken keresztül.

További információkért lásd az [ESLint Plugin](/docs/app/api-reference/config/next-config-js/eslint) oldalt.

## Abszolút importálások és modulútvonal-aliasok beállítása

A Next.js beépített támogatással rendelkezik a `tsconfig.json` és `jsconfig.json` fájlok `"paths"` és `baseUrl"` beállításaihoz.

Ezek a beállítások lehetővé teszik a projektkönyvtárak abszolút elérési utakra való aliasozását, ami megkönnyíti és letisztultabbá teszi a modulok importálását. Például:

```jsx
// Before
import { Button } from '../../../components/button'

// After
import { Button } from '@/components/button'
```

Az abszolút importálás konfigurálásához adja hozzá a `baseUrl` konfigurációs opciót a `tsconfig.json` vagy `jsconfig.json` fájlhoz. Például:

```json filename="tsconfig.json or jsconfig.json"
{
  "compilerOptions": {
    "baseUrl": "src/"
  }
}
```

A `baseUrl` elérési út konfigurálása mellett a ``paths`` opciót is használhatod az ``alias`` modul elérési útjaihoz.

Például a következő konfiguráció a `@/components/*`-t `components/*`-ra képezi le:

```json filename="tsconfig.json or jsconfig.json"
{
  "compilerOptions": {
    "baseUrl": "src/",
    "paths": {
      "@/styles/*": ["styles/*"],
      "@/components/*": ["components/*"]
    }
  }
}
```

Az ``elérési utak`` mindegyike relatív a ``baseUrl` helyéhez képest.
