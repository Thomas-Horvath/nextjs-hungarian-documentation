## Mi is a Next.js?

A Next.js egy React keretrendszer full-stack alkalmazások építéséhez. A React Components segítségével felhasználói felületeket hozhatsz létre, a Next.js pedig további funkciókat  és optimalizálási lehetőségeket biztosít.

Automatikusan konfigurálja az alacsonyabb szintű eszközöket,mint például a csomagolóprogramokat és a fordítókat. Így inkább a terméked fejlesztésére és a gyors megvalósításra koncentrálhatsz.

Akár egyéni fejlesztő vagy, akár egy nagyobb csapat tagja, a Next.js segíthet interaktív, dinamikus és gyors React alkalmazások létrehozásában.

## App Router és Pages Router

A Next.js két különböző útvonalkezelő rendszert használ:

- **App Router**: Az újabb útvonalkezelő rendszert, amely támogatja az új React funkciókat, például a Server Components-t.
- **Pages Router**: Az eredeti útvonalkezelő rendszert, még mindig támogatott és fejlesztés alatt áll.

Az oldalsáv tetején egy legördülő menüt találsz, amellyel válthatsz az [App Router](/docs/app) és a [Pages Router](/docs/pages) dokumentumok között.

### React verziókezelés

Az App Router és a Pages Router eltérően kezeli a React verziókat:

- **App Router**: Beépítve a [React canary verzióit](https://react.dev/blog/2023/05/03/react-canaries) használja, amelyek tartalmazzák a React 19 minden stabil változását, valamint azokat az új funkciókat is, amelyeket a keretrendszerekben tesztelnek a következő React-kiadás előtt.

- **Pages Router**: A projekted `package.json` fájljában megadott React-verziót használja.

Ez a megközelítés biztosítja, hogy az új React-funkciók megbízhatóan működjenek az App Routerben, miközben megőrzi a visszafelé kompatibilitást a meglévő Pages Router-alapú alkalmazásokkal.
